3. Estructuras de datos no lineales

Se encarga de organizan la información de una forma multidimensional o jerárquica, lo que permite tener relaciones más complejas.

En la unidad previa revisamos las estructuras de datos lineales más utilizadas en el área de la computación, también, revisamos las características que tiene e hicimos una comparativa de la complejidad de sus operaciones para que en determinado momento el alumno pueda decidir cuál de ellas le es más util para resolver un determinado problema. Ahora, vamos a revisar las estructuras de datos que no son lineales, como son los árboles, los montículos y los grafos, los cuales definen un orden jerárquico para almacenar la información, entonces que ventajas o desventajas nos puede acarrear esta forma de organización y para hacer esto estudiaremos como los podemos implementar en un programa computacional, además de analizar la complejidad de las operaciones y revisar algunas de sus implementaciones.

3.1
Árboles binarios

¿Alguna vez has visto o construido el árbol genealógico de tu familia?, o ¿has tenido oportunidad de ver el organigrama de una empresa? Habrás notado en ambos casos que la información se organiza por niveles, de forma jerárquica con base en ciertas reglas. Por ejemplo, para un árbol genealógico, piensa en un árbol invertido, los abuelos generalmente ocuparían el nivel más alto, es decir, en la raíz, los padres en el siguiente nivel y en el nivel más bajo, en las ramas, a los hijos. Pues bien, para poder tratar de forma computacional este tipo de disposición de la información, no basta con estructuras lineales como las que hemos visto hasta ahora, por lo que en este tema aprenderás nuevas formas de tratamiento de la información, que te permitirán trabajar con organizaciones de datos más dinámicas y complejas.

Comencemos con la definición de un árbol, sus elementos y características principales.

    Consideremos un árbol en general, los pasos por aplicar para lograr la conversión del árbol en general a un árbol binario son los siguientes:

    1. Deben enlazarse los hijos de cada nodo en forma horizontal, (los hermanos).

    2. Debe enlazarse en forma vertical el nodo padre con el hijo que se encuentra más a la izquierda; además debe eliminarse el vínculo de ese padre con el resto de sus hijos.

    3. Debe rotarse el diagrama resultante aproximadamente cuarenta y cinco grados hacia la izquierda; para obtener el árbol binario correspondiente.

    Insertar un nodo

    La operación de inserción en un árbol requiere una implementación recursiva, puesto que cada vez que se inserta un nuevo nodo, se debe descender por la estructura repetidamente hasta que, acorde a una regla de validación, se encuentre un nodo hoja, es decir un nodo que tenga disponible una referencia izquierda o derecha para poder enlazar el nuevo nodo.

    Veamos ahora la implementación, en ella, daremos prioridad a la inserción por la izquierda, observa cuidadosamente el proceso apoyándote de la explicación gráfica.

    Código 4
    /* Código en c para insertar un nodo */
void insertar(struct Nodo *nodo, int dato) {
 if (nodo->izquierda == NULL) {
 nuevo = nuevoNodo(dato);
 nodo->izquierda = nuevo;
 } 
 else if (nodo->derecha == NULL) {
 nuevo = nuevoNodo(dato);
 nodo->derecha = nuevo;
 } else {
 insertar(nodo->izquierda, dato);
 }
 }

    3.1.2
    Recorridos en un árbol binario

    Cuando se requiere acceder a los datos que se encuentran dentro de un árbol ya sea general o binario, se puede hacer mediante una secuencia lógica y metodológica de pasos, de tal forma que se puedan visitar todos los nodos y, por ende, procesar todos y cada uno de los datos, a esto se le denomina “recorrer” el árbol, de tal forma que si se está posicionado en un nodo cualquiera del árbol, mediante una operación de recorrido se puede: a. Descender a la rama izquierda, b. Descender a la rama derecha, o c. Procesar el dato (visitar el nodo).

    El recorrido entonces es una operación repetitiva que puede construirse de forma recursiva, para construir la secuencia para recorrer el árbol, toma en cuenta que, por convención, se recorre el subárbol izquierdo antes que el derecho, y el tipo de orden de recorrido, está determinado por las siguientes operaciones generales:

  • a. Preorden: Visitar el nodo antes de dirigirse al subárbol izquierdo. raíz -> rama izquierda -> rama derecha.
  • b. Inorden: Visitar el nodo después de recorrer el subárbol izquierdo, pero antes de recorrer el subárbol derecho. rama izquierda -> raíz -> rama derecha.
  • c. Postorden: Visitar el nodo después de recorrer ambos subárboles. rama izquierda -> rama derecha -> raíz.

    • Recorrido Preorden

    El recorrido Preorden consiste en: visitar primero la raíz del árbol-extraer dato, descender a la rama izquierda, si hay un nodo, descender al mismo – extraer dato, repetir la operación hasta que no existan más nodos en la rama izquierda (al encontrar null), cuando ocurra esto, subir un nivel y evaluar rama derecha, si hay un nodo, descender al mismo – extraer dato, repetir la operación y continuar hasta recorrer completamente el árbol.

    3.1.3
    Árbol binario de búsqueda

    Entre las operaciones básicas en cuanto a manejo de datos, se encuentra la búsqueda. Para abordar este problema, conocimos anteriormente, en la Unidad 1, algoritmos como el de búsqueda binaria, mediante el cual, se logra disminuir la complejidad de búsqueda de un elemento, sin embargo, esto aplica siempre y cuando los datos estuvieran ordenados, lo cual no siempre ocurre. Para resolver de una forma más eficiente esta problemática, se puede aprovechar la estructura de árbol binario, realizando algunos cambios en su implementación. Veamos sus características, construcción y utilidad.

    El Árbol Binario de Búsqueda es un caso especial del Árbol Binario, tiene la peculiaridad de realizar las inserciones de datos siguiendo procedimientos específicos, de tal forma que tanto el ordenamiento como la búsqueda de sus datos, se realice de una forma muy eficiente. Así, para construir un Árbol Binario de Búsqueda, el procedimiento más importante para el acomodo de los Nodos es tomar en cuenta el valor que esté almacenado dentro de ellos, por ejemplo, hablando de un árbol cuyos nodos contienen valores numéricos, aquellos nodos con valores menores al contenido en el nodo raíz, siempre se ubicarán en el subárbol izquierdo y a su vez, aquellos nodos con valores mayores al contenido en el nodo raíz, se colocarán en el subárbol derecho.

    3.1.7
    Nociones de complejidad de las implementaciones

    La complejidad logarítmica hace referencia a la cantidad de operaciones requeridas para ejecutar determinadas tareas computacionales, esto es, el tiempo que tarda un algoritmo basado en el tamaño de su entrada.

    En esta parte, exploraremos la complejidad de las operaciones de inserción, eliminación y búsqueda en estructuras de árbol binario de búsqueda, árbol equilibrado rojo-negro y montículo, considerando tanto el mejor como el peor caso.

    Árbol de Búsqueda Binario

    Recordando lo que se mencionó en la sección 3.1.3, el árbol binario de búsqueda es una estructura donde cada nodo tiene un valor asignado y a lo sumo tiene dos hijos, uno a la izquierda y otro a la derecha, y se ajusta a la propiedad de que el valor de cada nodo en el subárbol izquierdo (o hijo izquierdo) es menor que el valor de su nodo padre, mientras que el valor de cada nodo en el subárbol derecho (o hijo derecho) es mayor que el valor de su nodo padre. Esta característica se aplica de forma recursiva a todos los nodos del árbol.

    Complejidad en el escenario óptimo

    El caso óptimo en el árbol binario de búsqueda se presenta cuando este está perfectamente equilibrado, lo que indica que la altura del mismo es proporcional al logaritmo del total de nodos. Así, la complejidad para las operaciones de Búsqueda, Inserción y Eliminación es O(log n).

    Complejidad para el peor caso

    En el peor de los casos, el árbol puede ser considerado como una lista enlazada, lo que significa que cada nodo cuenta con un único hijo. Este escenario ocurre al agregar nodos de manera secuencial con valores ordenados que son ascendentes o descendentes, lo cual provoca que el árbol no esté equilibrado, resultando en un árbol cuya altura se iguala al total de nodos n. Por consiguiente, la complejidad de las operaciones de Búsqueda, Inserción y Eliminación es O(n).

    Árbol Balanceado Rojo-Negro

    Un árbol rojo-negro balanceado es una versión de árbol binario de búsqueda que garantiza que el árbol se mantenga equilibrado a través de reglas de color (rojo y negro). Estas normas aseguran que el árbol permanezca equilibrado con una altura que es logarítmica en función del número de nodos.

    Caso óptimo (Árbol equilibrado)

    En un árbol rojo-negro, el equilibrio del árbol se mantiene de manera estricta debido a sus normas de color. Su altura permanece constantemente logarítmica, garantizando que todas las operaciones fundamentales se efectúen en tiempo logarítmico.

    • Complejidad para Buscar, Insertar y Eliminar: O(log n)

    A pesar de que se hagan numerosas inserciones o eliminaciones, el árbol se ajusta automáticamente mediante rotaciones para conservar su equilibrio.

    Peor Escenario (Equilibrado por Naturaleza)

    El árbol rojo-negro siempre preserve un equilibrio, así que el caso más desfavorable sigue siendo O(log n). Independientemente de cuántas inserciones o eliminaciones se efectúen, el árbol nunca se desequilibra tanto como para llegar a una altura lineal.

    Montículo (Pila)

    El montículo (heap) es una forma de árbol completo que cumple con la característica de montículo. En un max heap, el valor de cada nodo es superior o igual al de sus hijos, mientras que en un min heap, el valor de cada nodo es inferior o igual al de sus hijos.

    Óptimo Escenario (Operaciones en Montículo)

    El escenario óptimo para un montículo se da cuando las acciones como la inserción o eliminación requieren una mínima cantidad de trabajo de “flotación” (heapify-up o heapify-down).

    • Complejidad para Insertar y Eliminar: O(log n)

    La inserción consiste en situar el nuevo elemento en la posición final y luego “ascender” para recuperar la propiedad del heap. La eliminación del valor máximo (en un max heap) o mínimo (en un min heap) consiste en trasladar el último nodo a la raíz y, posteriormente, “hundirlo” hacia abajo.

    Peor Escenario (Montículo Desequilibrado)

    En un montículo, el peor escenario para la inserción y eliminación continúa siendo O(log n), ya que cada operación de subida o bajada se lleva a cabo a lo largo de la altura del árbol, y la altura del árbol es siempre logarítmica (ya que se trata de un árbol completo).

    3.1.8
    Otros tipos de árboles

    Árbol B

    El Árbol B es una forma de organización de datos que se ajusta automáticamente y es un tipo de árbol de búsqueda que se usa principalmente para manejar grandes cantidades de datos de manera efectiva. Es comúnmente utilizado en bases de datos y sistemas de archivos, donde es necesario llevar a cabo operaciones rápidas de lectura y escritura, particularmente en dispositivos de almacenamiento externo como los discos duros.

    Un Árbol B es un tipo de árbol de búsqueda equilibrado, donde cada nodo puede poseer más de dos hijos y guarda varias claves. La principal característica que lo distingue de los árboles binarios convencionales es que cada nodo no cuenta únicamente con un par de hijos, sino que puede tener múltiples, y las claves en los nodos permanecen ordenadas.

    Características del Árbol B:

    • Orden m del árbol: El árbol B cuenta con un parámetro conocido como orden, representado como m, que establece cuántas claves y cuántos hijos puede contener un nodo:
      • Un nodo en un árbol B de grado m puede albergar hasta m-1 claves.
      • Un nodo puede poseer hasta m hijos (referencias a nodos hijos).
    • Raíz:
      • La raíz puede contener entre 2 y m-1 claves, o ser un nodo con 0 claves (si el árbol está vacío).
    • Nodos internos:
      • Los nodos internos deben contener entre ⌈m/2⌉ y m-1 claves (es decir, un mínimo de la mitad de las claves posibles).
    • Nodos de hojas:
      • Los nodos hoja deben contener entre ⌈m/2⌉ y m-1 claves, y son los únicos nodos que almacenan los datos reales (valores).
    • Balanceo:
      • El árbol permanece equilibrado durante las acciones de inserción y eliminación. Esto implica que todas las hojas están a la misma profundidad, garantizando que las búsquedas se ejecuten en tiempo logarítmico.
    • Organización:
      • Las claves en un nodo se encuentran organizadas de forma ascendente, lo que facilita llevar a cabo búsquedas binarias en su interior para localizar la clave requerida.

    Las operaciones fundamentales de un árbol B son:

    • Inserción. La incorporación en un árbol B es un proceso ágil que se lleva a cabo siguiendo estos pasos:
      • Introducir en un nodo hoja: Si el nodo hoja tiene espacio (es decir, contiene menos de m-1 claves), la clave se coloca directamente en el nodo de manera ordenada.
      • División de nodos: Si un nodo hoja está completo (posee m-1 claves) y se intenta añadir una nueva clave:
        • El nodo se fracciona en dos nodos.
        • La clave del medio se eleva al nodo padre. Si el nodo padre también está completo, este procedimiento se repite de forma recursiva hasta alcanzar la raíz.
      • Crecimiento de la raíz: Al dividirse la raíz, se genera una nueva raíz, lo que eleva la altura del árbol.
    • Búsqueda. La exploración en un árbol B lleva a cabo los siguientes pasos:
      • Inicia en la raíz y se procura la clave en el nodo con una búsqueda binaria, dado que las claves se encuentran ordenadas.
      • Según la clave hallada, el próximo paso será:
        • Si la clave se encuentra en el nodo, la clave ha sido hallada.
        • Si la clave no se encuentra en el nodo, se sigue el puntero adecuado (hijo) en el nodo hacia el siguiente nivel, repitiendo el procedimiento en ese hijo.
      • Este procedimiento se repite hasta dar con la clave o alcanzar un nodo hoja donde la clave no se encuentra, lo que indica que el elemento no está en el árbol.
    • Eliminación. La supresión de una clave en un árbol B se lleva a cabo de forma eficiente y puede incluir diversas estrategias:
      • Suprimir de un nodo hoja: Si la clave se encuentra en un nodo hoja y el nodo posee bastantes claves (como mínimo ⌈m/2⌉), se retira de manera directa.
      • Unión y fusión de nodos: Si quitar una clave disminuye la cantidad de claves en un nodo por debajo del mínimo (⌈m/2⌉), es preciso unir el nodo con un hermano cercano (un nodo hermano que posee más de ⌈m/2⌉ claves). Si esto no se puede hacer, el nodo relacionado puede fusionarse y se elimina la clave del nodo padre, lo que podría requerir que el nodo padre también se modifique de manera recursiva.

    Beneficios del Árbol B

    • Eficiencia en el almacenamiento en disco duro: Los árboles B son muy efectivos en almacenamiento ya que están diseñados para reducir al máximo el número de accesos al disco. Como cada nodo puede contener varias claves, la altura del árbol es baja, lo que disminuye los accesos al disco duro.
    • Búsquedas eficientes: Las búsquedas en un árbol B son eficientes (O(log n)) gracias a su estructura equilibrada, lo que facilita la localización de claves de manera ágil, incluso en grandes volúmenes de información.
    • Equilibrio automático: El árbol B se ajusta de forma automática durante los procesos de inserción y eliminación, asegurando que las búsquedas siempre se efectúen en tiempo logarítmico.

    Aplicaciones Típicas

    • Sistemas de bases de datos: Los árboles B forman la estructura principal de numerosos sistemas de bases de datos para clasificar e indexar grandes volúmenes de información. Su diseño mejora la lectura y escritura desde el disco, lo cual es esencial para bases de datos con altos volúmenes de registros.
    • Sistemas de archivos: Los sistemas de archivos actuales también emplean estructuras de árbol B para clasificar los archivos y carpetas en disco.
    • Almacenamiento y recuperación efectiva: Son perfectos para aplicaciones que requieren insertar, eliminar y consultar datos rápidamente, como motores de búsqueda, bases de datos y aplicaciones de análisis de grandes cantidades de información.

    Árbol B+

    El Árbol B+ representa una modificación del Árbol B, utilizado principalmente en sistemas de bases de datos y archivos, gracias a sus propiedades que optimizan la eficacia de las búsquedas y el acceso a la información. Su configuración está elaborada para maximizar tanto las funciones de búsqueda como las de recorrido secuencial.

    El Árbol B+ es una estructura de búsqueda equilibrada, en la que:

    • Únicamente las hojas albergan los valores de los datos (o registros completos).
    • Los nodos internos únicamente guardan claves (y referencias a nodos hijos), pero no tienen los valores vinculados a las claves.
    • Una de las principales ventajas del Árbol B+ en comparación con el Árbol B es que las hojas están conectadas, lo que permite un recorrido secuencial eficiente de los datos.

    Estructura de un Árbol B+

    Nodos interiores:

    • Claves: Los nodos intermedios únicamente tienen claves (denominadas índices), que ayudan a dirigir la búsqueda en el árbol. No incluyen los datos verdaderos, solo las referencias a los nodos secundarios.
    • Punteros: Cada clave en un nodo interno posee un puntero que se dirige a un subárbol hijo. Estos indicadores son los que facilitan la navegación por el árbol.

    Nodos de la hoja:

    • Claves y valores: En un Árbol B+, las hojas almacenan tanto las claves como los valores vinculados. Es decir, cada página no solo incluye las claves, sino que también guarda los datos reales o enlaces a los registros.
    • Conexiones entre hojas: Las hojas están conectadas de manera secuencial, lo que implica que cada hoja posee un puntero al siguiente nodo hoja. Esto permite que el trayecto de los datos en secuencia sea sumamente eficaz.

    Características principales del Árbol B+

    • Balanceado: Similar al Árbol B, el Árbol B+ también es un árbol balanceado, lo que implica que todos los nodos hoja se encuentran a la misma profundidad. Esto asegura que las búsquedas se efectúen siempre en tiempo logarítmico.
    • Mayor eficacia en las búsquedas: Gracias a su diseño y disposición, las búsquedas son veloces. Dado que los nodos internos solo poseen claves y carecen de datos, las búsquedas se llevan a cabo de forma eficiente sin necesidad de acceder a los datos reales hasta alcanzar las hojas.
    • Recorrido secuencial eficaz: Debido a que las hojas están conectadas, el acceso secuencial a los datos es muy ágil, lo que simplifica tareas como consultas de rango o la iteración a través del conjunto de datos.
    • No existen datos duplicados en los nodos internos: A diferencia del Árbol B, donde los nodos internos pueden incluir datos, el Árbol B+ asegura que los nodos internos solo contengan claves, lo que aumenta la eficiencia del árbol en almacenamiento y búsqueda.

    Acciones en el Árbol B+

    • Inserción. El procedimiento de inserción en un Árbol B+ se asemeja al del Árbol B, aunque presenta algunas variaciones:
      • Incorporar en una hoja: Si la hoja en la que se debe añadir la nueva clave tiene espacio, se incluye directamente en la hoja de forma ordenada.
      • División de hojas: Si una hoja está completa y se requiere añadir una nueva clave, la hoja se parte en dos, y la clave del centro se eleva al nodo superior. Si el nodo padre igualmente está lleno, este proceso se repite de manera recursiva.
      • Crecimiento de la raíz: Al dividirse la raíz, se forma una raíz nueva. Esto eleva la altura del árbol.
    • Búsqueda. La búsqueda en un Árbol B+ es eficaz gracias a su diseño:
      • Se inicia desde la raíz y se continua el puntero correspondiente en el nodo interno de acuerdo con las claves.
      • Al alcanzar un nodo hoja, se lleva a cabo una búsqueda secuencial en esa hoja.
      • Debido a que es un árbol balanceado, el tiempo necesario para la búsqueda es logarítmico en relación con la cantidad total de claves, lo que lo hace una estructura de datos muy eficiente.
    • Supresión. La supresión en un Árbol B+ se efectúa de forma análoga a la del Árbol B:
      • Eliminar de una hoja: Si se borra una clave de una hoja y esta mantiene suficientes claves, la eliminación se efectúa de manera directa.
      • Fusión de nodos: Si tras la eliminación, la hoja posee menos de la mitad de las claves requeridas, se puede combinar con un nodo hermano o tomar prestadas claves de un hermano adyacente para preservar el equilibrio del árbol.
      • Modificación en los nodos padres: Si la eliminación provoca que el nodo padre pierda una clave, el procedimiento puede extenderse hacia arriba en el árbol.

    Beneficios del Árbol B+

    Eficiencia en búsquedas secuenciales: Gracias a la conexión entre las hojas, el Árbol B+ facilita un recorrido secuencial eficaz. Esto resulta particularmente beneficioso para operaciones de consulta de rango, como cuando se quiere recuperar todos los elementos dentro de un rango de claves.

    Mejoramiento en el almacenamiento en disco duro: Debido a que las hojas incluyen los valores y están conectadas, el árbol maximiza la utilización de memoria y la cantidad de accesos al disco. Esto es fundamental en bases de datos y sistemas de archivos, donde los datos frecuentemente se encuentran guardados en dispositivos de almacenamiento externo.

    Gestión eficaz de grandes volúmenes de información: El Árbol B+ es ideal para administrar grandes cantidades de datos, como registros en bases de datos, ya que mantiene el árbol equilibrado y las búsquedas se realizan con eficiencia.

    Mejor manejo para consultas de rango: Debido a las hojas conectadas, el Árbol B+ resulta perfecto para efectuar consultas de rango o para recorrer todas las claves de manera efectiva, sin la necesidad de reexplorar el árbol.

    Bases de datos: El Árbol B+ se usa comúnmente en bases de datos relacionales para crear índices. Esto optimiza la rapidez de las búsquedas y las consultas de rango, dado que las hojas están organizadas y conectadas.

    Árbol de N elementos

    Un Árbol N-ario es una clase de estructura de datos donde cada nodo puede poseer hasta N hijos. Es una extensión de los árboles binarios, donde en los árboles binarios cada nodo puede tener un máximo de dos hijos (izquierdo y derecho). En un árbol N-ario, cada nodo puede tener hasta N descendientes, lo que proporciona mayor flexibilidad en la organización y estructuración de información. Esto indica que, a diferencia de los árboles binarios en los que un nodo puede tener hasta dos hijos, un árbol N-ario permite que un nodo tenga cualquier cantidad de hijos, siempre que no exceda el valor N.

    Características esenciales del Árbol N-ario:

    • Raíz: Al igual que cualquier árbol, el árbol N-ario cuenta con un nodo raíz que marca el comienzo del recorrido.
    • Hijos: Cada nodo puede tener hasta N hijos, los cuales son nodos hijos directos.
    • Niveles: Los nodos en el árbol se distribuyen en diferentes niveles. La raíz se encuentra en el nivel 0, y los niveles de los nodos secundarios incrementan según la profundidad de la estructura jerárquica.
    • Enlaces: Los nodos se relacionan mediante punteros, donde cada nodo dirige a sus descendientes. La cantidad de punteros en un nodo está determinada por el número de hijos que posee.

    Estructura de un Árbol N-ario

    Un Árbol N-ario se conforma por una secuencia de nodos, y cada nodo posee:

    • Valor (o clave): Datos guardados en el nodo.
    • Lista de hijos: Un conjunto (o arreglo) de punteros o referencias a los nodos hijos de dicho nodo. Si el nodo cuenta con menos de N hijos, algunos punteros podrían ser nulos.

    La raíz del árbol es el nodo inicial desde el cual se inician las operaciones de búsqueda o recorrido.

          A

B      C      D
/ \            / \
E   F          G   H

    En este caso:

    • El nodo principal es A, que tiene tres descendientes: B, C y D.
    • El nodo B cuenta con dos descendientes: E y F.
    • El nodo D cuenta con dos descendientes: G y H.
    • Los nodos C, E, F, G y H son hojas, lo que significa que no poseen hijos.

    Este es un Árbol N-ario con N = 3, donde cada nodo puede tener como máximo tres hijos.

    Acciones en un Árbol N-ario.

    Las acciones más frecuentes que se llevan a cabo en un Árbol N-ario son inserciones, búsquedas, eliminaciones y recorridos:

    Inserción. La inserción en un Árbol N-ario se lleva a cabo, por lo general, de la siguiente forma:

    • Añadir un hijo: Para incorporarlo a un árbol, primero localizamos el nodo donde deseamos insertar un hijo. Si el nodo cuenta con menos de N hijos, simplemente incorporamos el nuevo nodo a la lista de hijos.
    • División del nodo: Si un nodo cuenta con N hijos y se desea añadir otro, se puede dividir el nodo o reestructurar el árbol para incorporar el nuevo nodo.

    Búsqueda. La exploración en un árbol N-ario se lleva a cabo de forma recursiva:

    • Inicia desde el nodo raíz y se explora en cada uno de los hijos del nodo.
    • Si el valor deseado está en un nodo, se retorna ese nodo. De no ser así, la búsqueda prosigue de forma recursiva en los nodos hijos.

    Eliminación. La supresión de un nodo en un Árbol N-ario normalmente se lleva a cabo mediante estos pasos:

    • Se intenta localizar el nodo que se desea eliminar.
    • Si el nodo es un nodo terminal (no tiene descendientes), se elimina de inmediato.
    • Si el nodo cuenta con hijos, el procedimiento puede volverse más complicado, dependiendo de la normativa de eliminación. Normalmente, se traslada el contenido del nodo hijo al nodo que ha sido eliminado o se reorganiza el árbol.

    Recorridos. Así como en otros árboles, se pueden llevar a cabo diferentes tipos de recorridos en un árbol N-ario:

    • Recorrido en profundidad (DFS): Se empieza en el nodo raíz, visitando cada nodo y explorando a sus hijos de manera recursiva antes de proceder al siguiente hermano.
    • Exploración en amplitud (BFS): Se recorren los nodos por niveles, comenzando con todos los nodos en el nivel 0 (la raíz), luego se visitan todos los nodos en el nivel 1 (los descendientes de la raíz), y así sucesivamente.

    Beneficios del Árbol N-ario

    • Versatilidad: El Árbol N-ario ofrece una mayor versatilidad en comparación con los árboles binarios, dado que un nodo puede poseer más de dos hijos, lo que facilita la representación de estructuras de datos más elaboradas, como directorios o árboles de decisión.
    • Eficiencia en la representación de relaciones jerárquicas: Los árboles N-arios son idóneos para representar estructuras jerárquicas más intuitivas, como las estructuras de directorios en un sistema de archivos o los árboles de decisiones.
    • Equilibrio y profundidad: Según el valor de N, el árbol puede ser menos profundo que un árbol binario, lo que puede optimizar la eficiencia en las operaciones de búsqueda y recorrido en ciertas circunstancias.

    Usos del Árbol N-ario

    El Árbol N-ario cuenta con múltiples aplicaciones en informática, especialmente en campos que necesitan representar jerarquías o estructuras con un número variable de hijos por nodo.

    • Sistemas de archivos: Los sistemas de archivos generalmente utilizan árboles N-arios para representar directorios y archivos, donde un directorio puede albergar múltiples archivos o subdirectorios.
    • Árboles de decisión: En situaciones de elección, los árboles N-arios se utilizan para ilustrar diversas alternativas que pueden conducir a distintos resultados.
    • Modelos de relaciones jerárquicas: El Árbol N-ario es perfecto para representar relaciones jerárquicas entre entidades. Por ejemplo, en bases de datos, un nodo puede simbolizar una entidad y sus descendientes pueden denotar sub-entidades asociadas.
    • Compresión de datos: Ciertos algoritmos de compresión emplean estructuras de árboles N-arios para ilustrar la jerarquía de los datos que se van a comprimir.
    • Representación de expresiones algebraicas: En ciertas representaciones de expresiones algebraicas complejas, se pueden utilizar árboles N-arios para ilustrar términos que cuentan con más de dos operandos.
