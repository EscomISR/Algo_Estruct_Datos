1.2 El problema del ordenamiento

La importancia del ordenamiento ayuda a desarrollar un pensamiento lógico más estructurado para resolver problemas de forma óptima y organizada.

Es cuando se requiere que los datos se encuentren ordenados y en su caso si se encuentran desordenados hay que ordenarlos de tal manera que al ordenarlos se estructuren según un criterio especifico, con ello permite mayor identificación en dicho proceso ya que cada dato contendrá internamente un identificador en cual ayudara a distinguirlo y se puede comparar con los demás y no existirá duplicidad.

Este identificador único puede estar como parte de los datos en donde se le puede aplicar reglas en específico, es por lo que se requiere tener reglas de asignación para que sean ordenados de acuerdo con su secuencia de instrucciones en forma ascendente o descendente o por el criterio que se requiera.

(Knuth, 1998).
1.2.1
Ordenamiento por inserción

Este algoritmo inicia considerando el primer elemento del arreglo como punto de referencia inicial, para que cada uno de los siguientes elementos se consideren como finales, esto permite comparar cada de uno de ellos para desplazarse cada uno hacia la posición que le corresponda para que con ello se puedan intercambiar hasta llegar a su lugar correspondiente.

Un ejemplo de este ordenamiento es cuando se realiza un orden de cartas en la mano, en donde primero se toma una carta y esta es la referencia de la segunda esto permite que las demás se coloquen en forma continua y ordenadas hasta llegar a la última

1.2.1
Ordenamiento por selección

Este algoritmo se estructura bajo la premisa de los elementos más pequeños o grandes de una determina lista o arreglo para que con ello los coloque en la posición correcta entre los valores de 0 al tamaño -1, esto lo realiza varias veces hasta que todos los datos de la lista se encuentren ordenados.

El siguiente ejemplo, establece el ordenamiento por selección en donde consiste en tomar una lista desordenada y buscar el elemento más pequeño para colocarlo en la primera posición. Después se busca el siguiente elemento más pequeño en el resto de la lista y se coloca en la segunda posición. Este procedimiento se repite avanzando una posición cada vez, comparando los elementos restantes y colocándolos en el lugar correcto. Arquitectura del sistema operativo

Su estructura y organización de los distintos componentes que se comunican entre los equipos de cómputo y los programas de aplicación garantiza el uso eficiente, seguro y coordinado de los recursos del sistema.

El propósito es establecer una distribución lógica de funciones, donde cada capa o módulo cumple una tarea específica y se comunica con las demás mediante interfaces bien definidas.

En un sistema operativo moderno, la arquitectura suele dividirse en capas de abstracción:

  • Capa de hardware, que incluye los dispositivos físicos como CPU, memoria, disco y periféricos.
  • Núcleo o kernel, encargado de gestionar los recursos principales (procesos, memoria, archivos y dispositivos) y de mantener el control total del sistema.
  • Llamadas al sistema y bibliotecas, que actúan como una interfaz entre el kernel y las aplicaciones de usuario, permitiendo el acceso controlado a las funciones del sistema.
